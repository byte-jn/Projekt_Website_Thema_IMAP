IMAP

Internet Message Access Protocol

Grundprinzip:

IMAP ist ein Protokoll zum Abrufen und Verwalten von E-Mails direkt auf dem Server. Es ermöglicht Clients den gleichzeitigen Zugriff auf dieselbe Mailbox, ohne dass die Nachrichten lokal dauerhaft gespeichert werden müssen.

Wikipedia Link

Synchronisation und Statusverwaltung:

Da E-Mails auf dem Server verbleiben, bleiben Statusinformationen wie "gelesen", "ungelesen" oder "markiert" stets synchron – unabhängig davon, von welchem Gerät aus auf das Postfach zugegriffen wird.

Wikipedia Link

Erweiterte Funktionen:

IMAP unterstützt nicht nur den simplen Abruf von Nachrichten, sondern auch das Verwalten von Ordnerstrukturen, das Durchführen von Suchvorgängen und das selektive Herunterladen von Inhalten (z. B. nur Kopfzeilen).

Wikipedia Link

Technische Details:

Das Protokoll arbeitet standardmäßig über den TCP-Port 143. Für gesicherte Verbindungen wird üblicherweise der Port 993 in Kombination mit SSL/TLS verwendet.

Wikipedia Link

Vorteile gegenüber POP3:

Anders als POP3, das meist die E-Mails vom Server herunterlädt und entfernt, lässt IMAP die Nachrichten auf dem Server liegen und ermöglicht so eine flexiblere und geräteübergreifende Verwaltung der E-Mails.
