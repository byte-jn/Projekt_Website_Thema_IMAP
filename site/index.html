IMAP

Internet Message Access Protocol

Grundprinzip:

IMAP (Internet Message Access Protocol) ist eines der wichtigsten Protokolle für den Zugriff auf E-Mails, da es eine effiziente Verwaltung und Synchronisation von Nachrichten über mehrere Geräte hinweg ermöglicht.

Im Gegensatz zu POP3, bei dem E-Mails heruntergeladen und danach oft vom Server gelöscht werden, verbleiben bei IMAP alle Nachrichten auf dem Mail-Server. Dies sorgt dafür, dass Nutzer jederzeit und von überall auf ihre gesamte E-Mail-Korrespondenz zugreifen können, unabhängig davon, ob sie ein Smartphone, Tablet, Laptop oder Desktop-Computer verwenden.

Funktionsweise und technischer Aufbau:

IMAP arbeitet nach dem Client-Server-Prinzip, wobei der E-Mail-Client (z. B. Outlook, Thunderbird, Apple Mail oder eine mobile App) mit dem Mail-Server kommuniziert, um eine synchronisierte Kopie der E-Mails bereitzustellen.

Wenn eine neue Nachricht eingeht, bleibt sie auf dem Server gespeichert, während der Client eine Vorschau oder eine vollständige Kopie abruft. Dies hat den Vorteil, dass Speicherplatz auf dem Endgerät gespart wird, da E-Mails nicht lokal gespeichert werden müssen.

Ein wesentlicher Aspekt von IMAP ist die bidirektionale Synchronisation. Änderungen, die ein Nutzer an einer E-Mail vornimmt, werden direkt auf dem Server gespeichert und sind dadurch auf allen verbundenen Geräten sichtbar. Wird eine E-Mail auf dem Smartphone als gelesen markiert, erscheint sie auch auf dem Laptop als gelesen. Ebenso werden gelöschte oder verschobene E-Mails auf allen Geräten entsprechend aktualisiert.

IMAP-Befehle und Kommunikation mit dem Server

IMAP nutzt standardisierte Befehle, um die Kommunikation zwischen Client und Server zu ermöglichen. Einige der wichtigsten Befehle sind: Login, Select Inbox, Fetch, Store, Expunge

  • Login: Authentifiziert den Nutzer mit Benutzername und Passwort
  • Select Inbox: Wählt das Postfach aus, in dem Nachrichten verwaltet werden
  • Fetch: Ruft eine bestimmte E-Mail oder Kopfzeileninformationen ab
  • Store: Speichert Statusänderungen wie "gelesen" oder "markiert"
  • Expunge: Löscht dauerhaft als gelöscht markierte E-Mails

Beispiel für eine typische IMAP-Session:

A001 LOGIN benutzername passwort

A002 SELECT INBOX

A003 FETCH 1 BODY[]

A004 LOGOUT

Diese einfache Kommunikation zeigt, wie ein Client eine Verbindung aufbaut, das Postfach auswählt, eine Nachricht abruft und sich wieder abmeldet.

Technische Details:

Das Protokoll arbeitet standardmäßig über den TCP-Port 143. Für gesicherte Verbindungen wird üblicherweise der Port 993 in Kombination mit SSL/TLS verwendet.

Vorteile gegenüber POP3:

Anders als POP3, das meist die E-Mails vom Server herunterlädt und entfernt, lässt IMAP die Nachrichten auf dem Server liegen und ermöglicht so eine flexiblere und geräteübergreifende Verwaltung der E-Mails.
